:::tip[Коротко]
Терминал нужен, чтобы запускать Python, Git и серверы БД. Аналитику хватает ~15 команд: перемещаться по папкам, смотреть файлы, искать по содержимому и связывать команды через `|`. Не нужно становиться сисадмином — нужен минимум.
:::

## Навигация и файлы

| Команда | Что делает |
|---------|------------|
| `pwd` | показать текущую папку |
| `ls` | список файлов (`ls -la` — со скрытыми и деталями) |
| `cd папка` | перейти в папку (`cd ..` — на уровень вверх) |
| `mkdir имя` | создать папку |
| `cp a b` | скопировать файл |
| `mv a b` | переместить/переименовать |
| `rm файл` | удалить файл (`rm -r папка` — папку) |

## Просмотр содержимого

| Команда | Что делает |
|---------|------------|
| `cat file` | вывести весь файл |
| `head -n 20 file` | первые 20 строк |
| `tail -n 20 file` | последние 20 строк |
| `less file` | листать файл (выход — `q`) |

## Поиск

```bash
grep "ошибка" log.txt      # строки, где встречается "ошибка"
grep -ri "todo" .          # рекурсивно, без учёта регистра
find . -name "*.csv"       # найти все csv в дереве папок
wc -l data.csv             # сколько строк в файле
```

## Перенаправление и пайпы

```bash
ls > files.txt             # записать вывод в файл (перезапишет)
ls >> files.txt            # дописать в конец
cat data.csv | grep "RU" | wc -l   # строки с RU и их количество
```

Символ `|` («пайп») передаёт вывод одной команды на вход другой — основа работы в терминале.

:::note[bash / zsh / PowerShell]
На macOS по умолчанию **zsh**, на Linux обычно **bash** — команды выше работают одинаково. На Windows встроенный **PowerShell** отличается синтаксисом; проще поставить Git for Windows и пользоваться **Git Bash**, где работают unix-команды.
:::

:::caution[rm не спрашивает дважды]
`rm` удаляет мимо корзины, без подтверждения и безвозвратно. Дважды проверяй путь, особенно с `rm -r`. Никогда не запускай `rm -rf /` и подобное.
:::
