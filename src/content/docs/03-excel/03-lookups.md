:::tip[Коротко]
Лукап — это «подтянуть значение из другой таблицы по ключу» (аналог `JOIN` в SQL). Современный выбор — **`XLOOKUP`**: проще и без ограничений старого `VLOOKUP`. Если `XLOOKUP` недоступен (старый Excel) — используй связку **`INDEX` + `MATCH`**.
:::

## Зачем это нужно

Есть таблица заказов с `customer_id` и отдельный справочник клиентов. Лукап подставляет имя клиента к каждому заказу по `id`. Без него пришлось бы искать вручную — невозможно на тысячах строк.

```text title="Справочник (G:H)"
id   name
1    Анна
2    Борис
3    Кира
```

## VLOOKUP — классика с оговорками

`VLOOKUP(что_ищем; где_ищем; номер_столбца; 0)`. Последний аргумент `0` (или FALSE) — точное совпадение, его почти всегда и нужно.

```text
=VLOOKUP(A2; G:H; 2; 0)     по id из A2 вернёт имя   → "Анна"
```

:::caution[Три грабли VLOOKUP]
1. **Ищет только вправо** — ключ должен быть в левом столбце диапазона. Влево искать не умеет.
2. **Номер столбца — число** — вставишь столбец в таблицу, и `3` начнёт указывать не туда.
3. **Забытый `0`** в конце включает «приблизительный поиск» и тихо возвращает мусор. Всегда ставь `0`.
:::

## XLOOKUP — современная замена

`XLOOKUP(что; где_искать; что_вернуть)` — отдельные диапазоны для ключа и результата, точное совпадение по умолчанию, умеет искать влево:

```text
=XLOOKUP(A2; G:G; H:H)                по id вернёт имя        → "Анна"
=XLOOKUP(A2; G:G; H:H; "не найдено")  с заглушкой при промахе
```

Нет хрупкого «номера столбца», читается понятнее. Если есть в твоей версии Excel/Sheets — бери его.

## INDEX + MATCH — когда нет XLOOKUP

`MATCH` находит позицию ключа, `INDEX` достаёт значение по этой позиции. Работает в любую сторону и не ломается от вставки столбцов:

```text
=INDEX(H:H; MATCH(A2; G:G; 0))        → "Анна"
```

`MATCH(A2; G:G; 0)` вернёт номер строки с нужным id, `INDEX(H:H; ...)` — имя из этой строки.

## Что выбрать

| Ситуация | Решение |
|----------|---------|
| Современный Excel 365 / Sheets | `XLOOKUP` |
| Старый Excel без XLOOKUP | `INDEX` + `MATCH` |
| Нужно искать влево | `XLOOKUP` или `INDEX/MATCH` (не `VLOOKUP`) |
| Ключ не найден — нужна заглушка | `XLOOKUP(...; "—")` или `IFERROR(...)` |

<details>
<summary>1. Почему `VLOOKUP` не может найти значение левее ключа?</summary>

`VLOOKUP` всегда ищет ключ в первом (левом) столбце диапазона и возвращает столбец правее по номеру. Влево он конструктивно не умеет. Решение — `XLOOKUP` или `INDEX/MATCH`, где диапазоны ключа и результата независимы.

</details>

<details>
<summary>2. Что вернёт VLOOKUP, если забыть последний аргумент `0`?</summary>

Включится приблизительный поиск (предполагает отсортированный список) и формула может молча вернуть неверное значение вместо ошибки. Поэтому для точного поиска всегда явно указывай `0` (FALSE).

</details>
