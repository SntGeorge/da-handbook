:::tip[Коротко]
Реальные данные грязные: лишние пробелы, дубли, «23 единицы» вместо числа, ФИО в одной ячейке. Базовый набор для очистки — **`TRIM`** (убрать пробелы), **Удаление дубликатов**, **Text-to-Columns / Flash Fill** (разбить столбцы) и текстовые функции `LEFT`/`MID`/`FIND`.
:::

## Зачем это нужно

Грязные данные ломают всё дальше по цепочке: `VLOOKUP` не находит "Анна " (с пробелом) ≠ "Анна", сумма не считается из-за текста среди чисел. Очистка — обязательный первый шаг любого анализа.

## Удаление дубликатов

Выдели диапазон → **Данные → Удалить дубликаты** → отметь столбцы, по которым считать строку дублем. Excel оставит первое вхождение.

:::caution[Дубликаты удаляются необратимо]
Команда сразу меняет данные. Перед удалением сделай копию листа или работай на копии — отменить можно только через Ctrl+Z сразу, потом уже нет.
:::

## Find & Replace и Text-to-Columns

- **Find & Replace** (`Ctrl + H`) — массовая замена: убрать `₽`, поменять `,` на `.`, вычистить лишний символ.
- **Text-to-Columns** (Данные → Текст по столбцам) — разбить один столбец на несколько по разделителю (`;`, пробел) или по фиксированной ширине. Классика для «Имя Фамилия» в одной ячейке.

## Flash Fill

**Flash Fill** (`Ctrl + E`) — Excel угадывает шаблон по твоему примеру. Напечатал в соседнем столбце «Анна» напротив «Анна Иванова», нажал `Ctrl + E` — он заполнит остальные имена сам. Магия для одноразовой чистки без формул.

## Текстовые функции

Когда нужна повторяемая (формульная) чистка:

| Функция | Что делает | Пример → результат |
|---------|-----------|---------------------|
| `TRIM` | убирает лишние пробелы | `TRIM(" Анна ")` → `Анна` |
| `CLEAN` | убирает непечатаемые символы | `CLEAN(текст)` |
| `UPPER`/`LOWER`/`PROPER` | регистр | `PROPER("анна")` → `Анна` |
| `LEFT`/`RIGHT` | n символов с края | `LEFT("RU-123";2)` → `RU` |
| `MID` | символы с позиции | `MID("RU-123";4;3)` → `123` |
| `FIND` | позиция подстроки | `FIND("-";"RU-123")` → `3` |

Комбинируя `MID` и `FIND`, вытаскивают части по разделителю:

```text
=MID(A2; FIND("-";A2)+1; 99)     всё после дефиса   "RU-123" → "123"
```

:::note[FIND vs SEARCH]
`FIND` чувствителен к регистру и не понимает подстановочные знаки, `SEARCH` — наоборот (регистр игнорирует, понимает `*`/`?`). Для большинства задач очистки удобнее `SEARCH`.
:::

<details>
<summary>1. VLOOKUP не находит "Москва", хотя она точно есть в справочнике. В чём дело?</summary>

Почти наверняка в одном из значений лишний пробел ("Москва " ≠ "Москва") или непечатаемый символ. Оберни ключ в `TRIM` (и при необходимости `CLEAN`) с обеих сторон, либо почисти исходный столбец.

</details>

<details>
<summary>2. В столбце «RU-123», «KZ-4500» нужно отдельно код страны и номер. Чем?</summary>

Быстро — **Text-to-Columns** с разделителем `-` или **Flash Fill**. Формулами: `LEFT(A2;2)` для кода и `MID(A2; FIND("-";A2)+1; 99)` для номера.

</details>
