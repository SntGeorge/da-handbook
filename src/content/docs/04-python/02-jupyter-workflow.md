:::tip[Коротко]
Jupyter-ноутбук — это документ из **ячеек**: пишешь код, выполняешь по одной (`Shift+Enter`), сразу видишь результат и графики рядом. Это стандартная среда аналитика: код, вывод и текстовые пояснения в одном файле `.ipynb`.
:::

## Зачем это нужно

Анализ — итеративный процесс: загрузил данные, посмотрел, поправил, пересчитал. В обычном скрипте пришлось бы каждый раз перезапускать всё; в ноутбуке выполняешь по кусочку и держишь результаты в памяти. Поэтому весь разведочный анализ живёт именно в ноутбуках.

## Запуск

- **В VS Code** — создай файл `.ipynb` и открой (см. [настройку](/01-setup/ide-setup/)). Самый удобный путь.
- **В браузере** — команда `jupyter lab` или `jupyter notebook` в терминале активированного [окружения](/01-setup/virtual-environments/).

## Ячейки: код и markdown

Два типа ячеек:

- **Code** — Python-код; результат последней строки печатается под ячейкой.
- **Markdown** — текст с разметкой (заголовки, списки, формулы) для пояснений.

```python
import pandas as pd
df = pd.read_csv("orders.csv")
df.head()        # таблица выведется прямо под ячейкой
```

`Shift+Enter` — выполнить ячейку и перейти к следующей.

## Магические команды

Спецкоманды Jupyter начинаются с `%` (строчные) или `%%` (на всю ячейку):

| Команда | Что делает |
|---------|------------|
| `%timeit код` | замерить время выполнения |
| `%%time` | время всей ячейки |
| `%matplotlib inline` | рисовать графики внутри ноутбука |
| `%who` | показать определённые переменные |
| `!pip install pandas` | выполнить команду оболочки (через `!`) |

## Полезные шорткаты

В командном режиме (нажми `Esc`, рамка ячейки синяя):

- `A` / `B` — вставить ячейку выше / ниже;
- `D D` — удалить ячейку;
- `M` / `Y` — переключить в markdown / code;
- `Shift+Enter` — выполнить.

## Хорошие практики

:::caution[Главная ловушка ноутбуков — порядок выполнения]
Ячейки можно запускать в любом порядке, и состояние «запоминается». Из-за этого ноутбук, который у тебя «работает», может сломаться при запуске сверху вниз. Периодически делай **Restart & Run All** — это проверяет, что всё воспроизводится с чистого листа.
:::

- **Одна ячейка — одна мысль**: загрузка, потом очистка, потом график. Не сваливай всё в одну.
- **Markdown-заголовки** между блоками — ноутбук читается как отчёт.
- **Импорты — в первой ячейке**, чтобы их было видно сразу.

## Notebooks в VS Code

VS Code открывает `.ipynb` нативно: те же ячейки, плюс автодополнение, отладчик и Git-диффы. Не забудь выбрать интерпретатор своего venv (правый верхний угол) — иначе не подхватятся установленные пакеты.
