:::tip[Коротко]
NumPy — библиотека быстрых числовых массивов (`ndarray`), на которой построен pandas. Главная идея — **векторные операции**: вместо цикла по элементам пишешь `arr * 2`, и это работает разом и быстро. Знать NumPy глубоко не нужно, но понимать broadcasting полезно — он всплывает в pandas.
:::

## Зачем это нужно

Колонка в pandas — это по сути массив NumPy. Когда пишешь `df["amount"] * 1.2`, под капотом работает векторная операция NumPy. Понимание основ объясняет, почему pandas быстрый и как он себя ведёт.

## ndarray vs list

```python
import numpy as np

lst = [1, 2, 3]
arr = np.array([1, 2, 3])

# список: умножение повторяет элементы
lst * 2      # [1, 2, 3, 1, 2, 3]

# массив: умножение поэлементное (векторное)
arr * 2      # array([2, 4, 6])
```

Массив хранит числа одного типа подряд в памяти — отсюда скорость на больших данных и удобная математика.

## Создание массивов

```python
np.array([1, 2, 3])          # из списка
np.zeros(5)                  # [0. 0. 0. 0. 0.]
np.ones((2, 3))              # матрица 2×3 из единиц
np.arange(0, 10, 2)          # [0 2 4 6 8]
np.linspace(0, 1, 5)         # 5 точек от 0 до 1
```

## Индексация и срезы

Как у списков, плюс булевы маски и двумерный доступ:

```python
a = np.array([10, 20, 30, 40, 50])
a[1:3]          # array([20, 30])
a[a > 25]       # array([30, 40, 50]) — булева маска
```

Булева маска (`a > 25` даёт массив `True/False`, которым отбирают элементы) — основа фильтрации и в pandas.

## Broadcasting

NumPy «растягивает» меньший массив до формы большего, чтобы операция применилась поэлементно:

```python
prices = np.array([100, 200, 300])
prices * 1.2            # array([120., 240., 360.]) — скаляр растянут
prices + np.array([1, 2, 3])   # array([101, 202, 303])
```

Именно благодаря broadcasting в pandas работает `df["price"] * 1.2` без всяких циклов.

## Универсальные функции и агрегации

```python
np.sqrt(arr)            # корень поэлементно
np.log(arr)             # логарифм

a.sum()    a.mean()     # сумма, среднее
a.min()    a.max()      # минимум, максимум
a.std()                 # стандартное отклонение
```

:::note[Когда NumPy, когда pandas]
Чистый NumPy нужен редко — для матричных вычислений, ML-фич, численных операций. Для табличных данных (со столбцами и именами) бери pandas: он удобнее и построен поверх NumPy.
:::

## Задачи для самопроверки

<details>
<summary>1. Чем `[1,2,3] * 2` отличается от `np.array([1,2,3]) * 2`?</summary>

Список повторится: `[1, 2, 3, 1, 2, 3]`. Массив умножится поэлементно (векторно): `array([2, 4, 6])`. В этом ключевая разница NumPy — арифметика применяется к каждому элементу.

</details>

<details>
<summary>2. Что делает `a[a > 25]`?</summary>

Это булева маска: `a > 25` даёт массив `True/False`, а индексация им оставляет только элементы, где `True`. Тот же приём — основа фильтрации в pandas.

</details>
