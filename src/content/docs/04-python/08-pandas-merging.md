:::tip[Коротко]
**`merge`** — это `JOIN` из SQL: склеить две таблицы по ключу (`inner`/`left`/`right`/`outer`). **`concat`** — сложить таблицы стопкой (по строкам, как `UNION`) или приставить столбцы. Главная ловушка — раздувание строк при дублях в ключе.
:::

## Зачем это нужно

Данные почти всегда разнесены по таблицам: заказы отдельно, клиенты отдельно. Чтобы к заказу подтянуть имя клиента или к продажам — справочник товаров, нужен `merge`.

```python title="Две таблицы"
orders = pd.DataFrame({
    "order_id":   [101, 102, 104],
    "customer_id":[1, 1, 3],
    "amount":     [2500, 1800, 4200],
})
customers = pd.DataFrame({
    "customer_id":[1, 2, 3],
    "name":       ["Анна", "Борис", "Кира"],
})
```

## merge: соединение по ключу

```python
orders.merge(customers, on="customer_id", how="left")
```

| order_id | customer_id | amount | name |
|----------|-------------|--------|------|
| 101      | 1           | 2500   | Анна |
| 102      | 1           | 1800   | Анна |
| 104      | 3           | 4200   | Кира |

## Типы соединений (how=)

| how | Что оставляет |
|-----|---------------|
| `inner` (по умолчанию) | только совпавшие в обеих |
| `left` | все строки левой + совпавшие из правой |
| `right` | все строки правой + совпавшие из левой |
| `outer` | все строки обеих, недостающее → NaN |

Если имена ключей разные — `left_on="id"`, `right_on="customer_id"`.

## join по индексу

`join` — это `merge`, но по индексу (короче, когда ключ — индекс):

```python
orders.set_index("customer_id").join(customers.set_index("customer_id"))
```

## concat: стопкой и встык

```python
pd.concat([jan, feb])                    # по строкам: выгрузки за разные месяцы (UNION)
pd.concat([df1, df2], axis=1)            # по столбцам: приставить рядом
```

`axis=0` (по умолчанию) — друг под другом; `axis=1` — бок о бок по индексу.

## combine_first

Заполнить пропуски в одной таблице значениями из другой:

```python
primary.combine_first(backup)            # где в primary NaN, возьмёт из backup
```

:::caution[Главная ошибка merge — раздувание строк]
Если ключ в правой таблице не уникален, каждая строка левой совпадёт с **несколькими** правыми, и число строк вырастет (а суммы задвоятся). Перед `merge` проверяй уникальность ключа: `customers["customer_id"].is_unique`. Это та же ловушка, что [«размножение строк» в SQL-джойнах](/02-sql/06-joins/).
:::

## Задачи для самопроверки

<details>
<summary>1. Нужны все заказы, и если у клиента есть имя — подтянуть его, иначе NaN. Какой how?</summary>

`how="left"` с таблицей заказов слева: `orders.merge(customers, on="customer_id", how="left")`. Сохранятся все заказы; имя подставится где есть, иначе будет `NaN`.

</details>

<details>
<summary>2. После merge сумма выручки внезапно выросла вдвое. Почему?</summary>

Скорее всего, ключ в правой таблице не уникален — строки левой размножились на совпадения. Проверь `df["key"].is_unique` и при необходимости агрегируй/дедуплицируй правую таблицу до merge.

</details>
