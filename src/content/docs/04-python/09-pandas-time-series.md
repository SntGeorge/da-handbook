:::tip[Коротко]
Сначала приведи столбец к дате через **`to_datetime`** — тогда откроются временные операции: **`resample`** (агрегировать по дням/неделям/месяцам), **`rolling`** (скользящее среднее), **`shift`/`diff`** (сравнение с прошлым периодом). Это основа отчётов «по месяцам» и «прирост к предыдущему».
:::

## Зачем это нужно

Аналитика во времени — каждый день: выручка по неделям, MAU по месяцам, рост к прошлому периоду, сглаживание шума скользящим средним. pandas делает это парой методов, но только если даты — настоящие даты, а не строки.

## to_datetime

```python
df["date"] = pd.to_datetime(df["date"])          # строка → дата
df["date"] = pd.to_datetime(df["date"], format="%d.%m.%Y")   # если формат нестандартный
```

После этого доступны компоненты: `df["date"].dt.year`, `.dt.month`, `.dt.dayofweek`, `.dt.to_period("M")`.

## DatetimeIndex

Многие операции удобнее, когда дата — это индекс:

```python
ts = df.set_index("date")
ts.loc["2026-02"]            # все строки за февраль 2026
ts.loc["2026-01":"2026-03"] # срез по диапазону дат
```

## resample: агрегация по периодам

`resample` — это `groupby` по времени. Коды периодов: `D` (день), `W` (неделя), `ME` (месяц), `QE` (квартал):

```python
ts["amount"].resample("ME").sum()       # выручка по месяцам
ts["amount"].resample("W").mean()        # средний чек по неделям
```

| период | сумма |
|--------|-------|
| 2026-01 | 4300 |
| 2026-02 | 4900 |

## rolling: скользящие окна

Сглаживание и скользящие метрики:

```python
ts["amount"].rolling(7).mean()           # 7-дневное скользящее среднее
ts["amount"].rolling(30).sum()           # сумма за последние 30 точек
```

Скользящее среднее убирает дневной шум и показывает тренд.

## shift и diff: сравнение с прошлым

```python
monthly = ts["amount"].resample("ME").sum()
monthly.shift(1)                         # значение прошлого месяца
monthly.diff()                           # абсолютный прирост к прошлому
monthly.pct_change()                     # прирост в процентах (month-over-month)
```

`shift` сдвигает ряд (аналог `LAG` в SQL), `diff` и `pct_change` сразу дают прирост.

:::caution[Сортируй по дате перед shift/rolling]
`shift`, `diff` и `rolling` работают по текущему порядку строк. Если данные не отсортированы по дате, «предыдущий период» окажется случайным. Перед расчётами делай `df.sort_values("date")`.
:::

## Таймзоны (кратко)

Если время в UTC, а нужен локальный пояс:

```python
ts.tz_localize("UTC").tz_convert("Europe/Moscow")
```

На старте таймзоны редко нужны — но помни, что «полночь» зависит от пояса, и это влияет на суточные метрики.

## Задачи для самопроверки

<details>
<summary>1. Как получить выручку по месяцам из таблицы с ежедневными заказами?</summary>

Привести дату: `df["date"] = pd.to_datetime(df["date"])`, сделать её индексом и ресемплить: `df.set_index("date")["amount"].resample("ME").sum()`. `resample` — это группировка по времени.

</details>

<details>
<summary>2. Чем `diff()` отличается от `pct_change()`?</summary>

`diff()` даёт абсолютную разницу с предыдущим значением (например, +500), `pct_change()` — относительную в долях (например, 0.12 = +12%). Для month-over-month в процентах берут `pct_change`.

</details>
