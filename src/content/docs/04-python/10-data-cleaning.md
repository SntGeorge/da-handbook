:::tip[Коротко]
Перед анализом данные нужно почистить: разобраться с **пропусками** (`isna`, `fillna`, `dropna`), убрать **дубликаты** (`drop_duplicates`), привести **типы** (`astype`, `to_numeric`) и решить, что делать с **выбросами**. На грязных данных любые выводы недостоверны.
:::

:::note[Поток данных]
Вход: сырая «грязная» выгрузка (пропуски, дубли, числа-как-текст)
→ Обработка: `fillna`/`dropna`, `drop_duplicates`, `to_numeric`, обработка выбросов
→ Выход: чистый DataFrame, готовый к анализу.
Зачем: гарантировать, что выводы строятся на корректных данных, а не на мусоре.
:::

## Зачем это нужно

Реальные выгрузки всегда грязные: пустые ячейки, дубли строк, число записано как текст, опечатки в категориях. Очистка — обязательный этап; на него уходит до половины времени анализа, и это нормально.

## Пропуски (NaN)

```python
df.isna().sum()                    # сколько пропусков в каждом столбце
df.dropna()                        # удалить строки с любым NaN
df.dropna(subset=["amount"])       # только если NaN в amount
df["amount"].fillna(0)             # заполнить нулём
df["amount"].fillna(df["amount"].median())   # медианой
df["country"].fillna("unknown")    # категориальные — заглушкой
```

:::caution[Удалять или заполнять?]
`dropna` теряет данные — оправдан, когда пропусков мало или строка без ключа бесполезна. `fillna` сохраняет строки, но вносит предположение (медиана/0/«unknown»). Выбор зависит от смысла: пропущенную сумму нулём заполнять опасно, пропущенную категорию «unknown» — обычно нормально.
:::

## Дубликаты

```python
df.duplicated().sum()              # сколько полных дублей
df.drop_duplicates()               # убрать полные дубли
df.drop_duplicates(subset=["order_id"], keep="last")   # по ключу, оставить последний
```

## Приведение типов

Частая проблема — число или дата прочитались как текст:

```python
df["amount"] = df["amount"].astype(float)
df["amount"] = pd.to_numeric(df["amount"], errors="coerce")   # нечисла → NaN
df["date"]   = pd.to_datetime(df["date"])
df["count"]  = df["count"].astype("Int64")    # целые с поддержкой NaN
```

`to_numeric(errors="coerce")` спасает, когда среди чисел затесался мусор: он превратит его в `NaN`, а не уронит весь столбец.

## replace: правка значений

```python
df["status"] = df["status"].replace({"paid ": "paid", "PAID": "paid"})   # привести к единому виду
df["country"] = df["country"].str.strip().str.upper()                    # убрать пробелы, регистр
```

## Выбросы

Найти аномальные значения, например по правилу межквартильного размаха (IQR):

```python
q1, q3 = df["amount"].quantile([0.25, 0.75])
iqr = q3 - q1
mask = df["amount"].between(q1 - 1.5*iqr, q3 + 1.5*iqr)
df_clean = df[mask]                # оставить «нормальные»
```

Выброс — не всегда ошибка: крупный заказ может быть настоящим. Решай по смыслу, не удаляй вслепую.

## Категориальные данные

Для столбцов с повторяющимися значениями (страна, статус) тип `category` экономит память и ускоряет группировки:

```python
df["country"] = df["country"].astype("category")
```

<details>
<summary>1. Столбец amount прочитался как текст из-за пробелов и «—». Как привести к числу?</summary>

`pd.to_numeric(df["amount"], errors="coerce")` — превратит непарсящиеся значения («—», пустые) в `NaN`, остальное в числа. Затем при необходимости `fillna` или `dropna`. Просто `astype(float)` упал бы на «—».

</details>

<details>
<summary>2. Когда выброс стоит удалить, а когда оставить?</summary>

Удалять — если это явная ошибка (отрицательный возраст, сумма в 1000× из-за бага). Оставлять — если значение реальное, пусть и редкое (крупный клиент). Слепое отсечение по IQR может выкинуть ценные наблюдения, поэтому смотри на природу данных.

</details>
