:::tip[Коротко]
Plotly строит **интерактивные** графики: наведение с подсказками, зум, фильтрация прямо в браузере. Быстрый путь — **`plotly.express`** (`px`), одна строка на график; тонкий контроль — `graph_objects`. Результат сохраняется в самодостаточный HTML — удобно делиться без скриншотов.
:::

:::note[Поток данных]
Вход: DataFrame
→ Обработка: `px.<тип>(df, ...)` строит интерактивный график
→ Выход: интерактивный график в ноутбуке или самодостаточный HTML-файл.
Зачем: дать возможность «покрутить» данные (hover, zoom) и поделиться без скриншотов.
:::

## Зачем это нужно

Статичный график ([matplotlib](/04-python/11-matplotlib-seaborn/)) хорош для отчёта-картинки. Но когда нужно «покрутить» данные — навести на точку, приблизить период, скрыть серию — нужна интерактивность. Plotly даёт её почти даром и встраивается в дашборды (Streamlit, Dash).

## plotly.express: быстрый старт

`px` принимает DataFrame и строит график одной строкой:

```python
import plotly.express as px

fig = px.bar(df, x="country", y="amount", color="status", title="Выручка")
fig.show()

px.line(df, x="date", y="amount")              # линия во времени
px.scatter(df, x="amount", y="net", color="country", hover_data=["order_id"])
px.histogram(df, x="amount", nbins=20)
```

Параметр `color` автоматически разбивает данные по категориям и строит легенду.

## graph_objects: полный контроль

Когда `express` не хватает (несколько осей, нестандартная компоновка) — собирают график вручную:

```python
import plotly.graph_objects as go

fig = go.Figure()
fig.add_trace(go.Scatter(x=df["date"], y=df["amount"], mode="lines", name="Выручка"))
fig.update_layout(title="Динамика", xaxis_title="Дата", yaxis_title="₽")
```

Обычно начинают с `px`, а к `graph_objects` переходят только при необходимости.

## Интерактивность

Из коробки, без кода: **hover** (подсказка при наведении), **zoom** (выделить область), **pan** (перетаскивание), **клик по легенде** (скрыть/показать серию), кнопка скачать PNG. Настроить подсказку — параметром `hover_data`.

## Сохранение в HTML

```python
fig.write_html("chart.html")                   # самодостаточный интерактивный файл
fig.write_image("chart.png")                   # статичная картинка (нужен kaleido)
```

HTML открывается в любом браузере и сохраняет всю интерактивность — удобно отправить коллеге.

:::note[Когда Plotly, когда matplotlib/Seaborn]
Plotly — для интерактивных дашбордов и презентаций, где данные хочется исследовать. matplotlib/Seaborn — для статичных картинок в отчёт/PDF и быстрого EDA. Для научной точной графики чаще берут matplotlib, для «живых» отчётов — Plotly.
:::

## Plotly в Jupyter

В ноутбуке `fig.show()` рендерит интерактивный график прямо под ячейкой. В VS Code это работает из коробки; в классическом Jupyter иногда нужен пакет `nbformat`.

<details>
<summary>1. Нужен график выручки по странам с разбивкой по статусу и подсказками. Чем?</summary>

`px.bar(df, x="country", y="amount", color="status")`. Параметр `color="status"` сам разобьёт столбцы по статусам и добавит легенду, а hover с суммами включён по умолчанию.

</details>

<details>
<summary>2. Как поделиться интерактивным графиком с коллегой без Python?</summary>

`fig.write_html("chart.html")` — получится самодостаточный файл, который открывается в любом браузере с сохранением зума и подсказок. Python для просмотра не нужен.

</details>
