:::tip[Коротко]
Доверительный интервал (ДИ) — диапазон, в котором, скорее всего, лежит истинное значение параметра. **95% ДИ означает: если повторить эксперимент много раз, 95% таких интервалов накроют истину** — это свойство метода, а не вероятность для конкретного интервала. Понимание этого отличает аналитика от «рисователя графиков».
:::

## Зачем это нужно

Точечная оценка («конверсия 5%») не говорит, насколько мы уверены. ДИ добавляет границы погрешности («5% ± 1%»), и это меняет решения: интервал [4.9%, 5.1%] и [1%, 9%] — совсем разная уверенность при одной и той же оценке 5%.

## Что значит 95% ДИ на самом деле

:::caution[Самая частая ошибка интерпретации]
**Неверно:** «истинное среднее лежит в этом интервале с вероятностью 95%».
**Верно:** «если повторить выборку много раз и каждый раз строить такой интервал, 95% из них накроют истинное среднее».

Истинный параметр фиксирован, случаен — интервал. Конкретный посчитанный интервал либо накрыл истину, либо нет; 95% — это надёжность **процедуры**, а не вероятность для одного интервала. На собеседовании это любят проверять.
:::

## Расчёт для среднего

$$\bar{x} \pm z \cdot \frac{\sigma}{\sqrt{n}}$$

где $z = 1.96$ для 95%. Ширина зависит от разброса $\sigma$ и размера выборки $n$ (через [стандартную ошибку](/05-statistics/04-clt-and-sampling/)).

**На числах.** Берём пример [со страницы про SE](/05-statistics/04-clt-and-sampling/): $\bar{x}=300$ сек, $\sigma=120$, $n=400$, значит $SE=6$.

$$300 \pm 1.96 \cdot 6 = 300 \pm 11.76 = [288.2;\ 311.8] \text{ сек}$$

Читается так: «истинное среднее время на сайте по всем пользователям, скорее всего, между 288 и 312 секундами». Откуда `1.96`: это $z$-значение, отсекающее центральные 95% нормального распределения (для 90% берут 1.64, для 99% — 2.58).

```python
from scipy import stats
import numpy as np

data = df["amount"]
mean, se = data.mean(), stats.sem(data)
stats.t.interval(0.95, len(data)-1, loc=mean, scale=se)   # (нижняя, верхняя)
```

На практике берут t-распределение (а не $z$), особенно при малых выборках.

## Расчёт для доли

Для конверсий и долей:

$$\hat{p} \pm z \sqrt{\frac{\hat{p}(1-\hat{p})}{n}}$$

**На числах.** 50 покупок из 1000 → $\hat{p}=0.05$.

$$0.05 \pm 1.96 \sqrt{\frac{0.05 \cdot 0.95}{1000}} = 0.05 \pm 1.96 \cdot 0.0069 = 0.05 \pm 0.0135 = [3.6\%;\ 6.4\%]$$

Практический вывод: конверсия «5%» на самом деле — это «где-то между 3.6% и 6.4%». Поэтому сравнивать две конверсии «на глаз» (5.0% vs 5.2%) бессмысленно, если их интервалы перекрываются, — для этого и нужен [A/B-тест](/09-ab-testing/01-fundamentals/). Чем больше $n$, тем уже интервал: на 100 000 показов тот же $\hat{p}=0.05$ даст ДИ ≈ [4.9%, 5.1%].

## Bootstrap

Когда формулы нет или распределение странное — **bootstrap**: многократно берём выборку с возвращением, считаем статистику, смотрим на её разброс:

```python
import numpy as np
boot = [np.random.choice(data, len(data), replace=True).mean() for _ in range(10000)]
np.percentile(boot, [2.5, 97.5])          # 95% ДИ из перцентилей
```

Универсальный метод: работает для медианы, перцентилей и любых сложных метрик.

## Типичные ошибки

- Трактовать 95% как вероятность для конкретного интервала (см. выше).
- Думать, что 95% данных лежат в ДИ — нет, ДИ про **параметр** (среднее), а не про отдельные значения.
- Сравнивать группы «на глаз по перекрытию ДИ» вместо [теста гипотез](/05-statistics/06-hypothesis-testing/) — перекрытие не всегда означает отсутствие значимого различия.

<details>
<summary>1. «95% ДИ для среднего = [100, 120]». Верно ли «среднее лежит здесь с вероятностью 95%»?</summary>

Строго — нет. Истинное среднее фиксировано: оно либо в [100, 120], либо нет. 95% относится к процедуре: при многократном повторении 95% построенных интервалов накроют истину. Это про надёжность метода, а не про конкретный интервал.

</details>

<details>
<summary>2. Как сузить доверительный интервал?</summary>

Увеличить размер выборки $n$ (ширина ∝ $1/\sqrt{n}$) либо снизить уровень доверия (90% уже, чем 95%, но рискованнее). Уменьшить разброс данных $\sigma$ обычно нельзя — он свойство популяции. Основной рычаг — больше данных.

</details>
