:::tip[Коротко]
Тест выбирается по типу данных и числу групп: для **средних двух групп** — t-test (или Mann-Whitney, если данные не нормальны), для **категориальных** — chi-square, для **нескольких групп** — ANOVA. Главное — не лепить t-test везде: на ненормальных или категориальных данных он даёт неверные выводы.
:::

## Зачем это нужно

Проверка гипотез ([предыдущая страница](/05-statistics/06-hypothesis-testing/)) даёт логику, а тест — конкретный инструмент под твои данные. Применить не тот тест — получить ложное p-value и неверное решение.

## t-test: сравнение средних

Проверяет, различаются ли средние. Три варианта:

| Вид | Когда |
|-----|-------|
| **Одновыборочный** | среднее выборки vs известное значение |
| **Двухвыборочный** | средние двух независимых групп (A vs B) |
| **Парный** | до/после на одних и тех же объектах |

```python
from scipy import stats
stats.ttest_ind(group_a, group_b)        # двухвыборочный
stats.ttest_rel(before, after)           # парный
```

Предполагает приближённую **нормальность** и (классически) сравнимые дисперсии.

## Проверка нормальности

t-test и ANOVA опираются на нормальность — её стоит проверить:

```python
stats.shapiro(data)         # Shapiro-Wilk: p < 0.05 → НЕ нормально
```

Плюс гистограмма и Q-Q plot глазами. На больших выборках тесты нормальности слишком чувствительны — смотри и на график.

## Mann-Whitney U: непараметрический аналог

Если данные не нормальны или это ранги/порядковые — сравнивай распределения через Mann-Whitney U вместо t-test:

```python
stats.mannwhitneyu(group_a, group_b)
```

Он сравнивает не средние, а сдвиг распределений; устойчив к выбросам и перекосу — частый выбор для денег и длительностей.

## chi-square: для категориальных

Проверяет связь между категориальными признаками (например, «зависит ли покупка от страны») по таблице сопряжённости:

```python
import pandas as pd
table = pd.crosstab(df["country"], df["converted"])
stats.chi2_contingency(table)
```

## ANOVA: несколько групп

Когда групп **больше двух** (A/B/C/D), не гоняй t-test попарно (растёт [ошибка I рода](/05-statistics/06-hypothesis-testing/)). ANOVA проверяет «есть ли различие хотя бы между какими-то группами» одним тестом:

```python
stats.f_oneway(group_a, group_b, group_c)
```

Непараметрический аналог — Kruskal-Wallis.

## Шпаргалка: какой тест когда

| Задача | Данные | Тест |
|--------|--------|------|
| 2 средних, нормально | числовые | t-test |
| 2 группы, не нормально | числовые/ранги | Mann-Whitney U |
| До/после на тех же объектах | числовые | парный t-test |
| Связь категорий | категориальные | chi-square |
| 3+ групп, нормально | числовые | ANOVA |
| 3+ групп, не нормально | числовые | Kruskal-Wallis |

:::caution[Не применяй t-test попарно к многим группам]
Сравнивая 4 группы всеми парами (6 тестов) при $\alpha=0.05$, ты раздуваешь вероятность ложной находки. Используй ANOVA, а уже при значимом результате — попарные сравнения с поправкой (например, Tukey HSD).
:::

<details>
<summary>1. Сравниваешь время на сайте (сильно перекошено) в двух группах. t-test или нет?</summary>

Лучше Mann-Whitney U: время на сайте обычно ненормально (длинный правый хвост), а t-test предполагает нормальность. Mann-Whitney устойчив к перекосу и выбросам. Либо примени t-test к лог-преобразованным данным, если они после этого нормализуются.

</details>

<details>
<summary>2. Тестируешь 4 варианта лендинга. Почему не 6 попарных t-test?</summary>

Шесть сравнений при $\alpha=0.05$ сильно повышают шанс ложноположительного результата (проблема множественных сравнений). Правильно — сначала ANOVA (есть ли различие в принципе), затем при значимости попарные тесты с поправкой.

</details>
