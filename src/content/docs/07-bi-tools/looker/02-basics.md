:::tip[Коротко]
В Looker Studio дашборд собирается из визуалов перетаскиванием полей, метрики-формулы — через **calculated fields**, а интерактив дают **controls** (выпадающие фильтры, выбор дат). Шаринг — как у Google-документа. Выбирай его для бесплатных/быстрых/Google-ориентированных задач, а Tableau/Power BI — для тяжёлых корпоративных.
:::

## Зачем это нужно

Зная базовый цикл Looker Studio, ты за вечер собираешь живой дашборд для пет-проекта и кладёшь ссылку в резюме. Это самый дешёвый способ показать навык дашбординга.

## Создание дашборда

1. **Create → Report**, подключи источник.
2. Добавляй визуалы (Add a chart): scorecard для KPI, time series для динамики, table, bar.
3. Кидай поля в **Dimension** и **Metric** визуала.
4. Оформи: тема, заголовки, расположение. Режим **View** показывает, как увидит зритель.

## Calculated fields

Новые метрики из формул прямо в отчёте (функции в духе SQL/таблиц):

```
-- маржа
Revenue - Cost

-- категория суммы
CASE WHEN Amount > 2000 THEN "крупный" ELSE "мелкий" END
```

Создаются на уровне источника или конкретного поля. Аналог [вычисляемых полей Tableau](/07-bi-tools/tableau/03-calculated-fields/).

## Filters и controls

- **Filter** (на уровне визуала/страницы) — жёстко ограничивает данные, зритель не меняет.
- **Control** — интерактивный виджет для зрителя: drop-down list, date range picker, поиск. Зритель сам крутит дашборд.

Для дашборда «выбери период и регион» добавляешь date range control и list control — данные перестраиваются.

## Шаринг и embed

Как у Google Docs: кнопка **Share** → по ссылке/по email/публично, с правами View или Edit. Готовый отчёт можно встроить (embed) на сайт через iframe.

:::caution[Доступ по ссылке = публичность]
Открыв доступ «всем, у кого есть ссылка», ты делаешь данные доступными любому, кто получит URL. Для отчётов с чувствительными данными выдавай доступ конкретным аккаунтам, а не публичной ссылкой. Это тот же риск, что у любого расшаренного Google-документа.
:::

## Когда выбирать Looker Studio

| Бери Looker Studio | Бери Tableau / Power BI |
|--------------------|-------------------------|
| бесплатно, пет-проект, портфолио | корпоративные требования |
| данные в экосистеме Google | сложная модель, тяжёлый DAX/LOD |
| быстрый дашборд, шаринг ссылкой | большие объёмы, RLS, on-premise |
| работа с любой ОС в браузере | продвинутые вычисления |

## Задачи для самопроверки

<details>
<summary>1. Нужно, чтобы зритель сам выбирал период и регион на дашборде. Filter или control?</summary>

Control: добавить date range control и list control (по региону) — это интерактивные виджеты, которыми управляет зритель. Filter жёстко ограничивает данные и зрителю недоступен для изменения. Для самостоятельного «кручения» дашборда нужны именно controls.

</details>

<details>
<summary>2. Когда Looker Studio предпочтительнее Power BI?</summary>

Когда нужен бесплатный быстрый дашборд, данные в Google-экосистеме (Sheets, GA4, BigQuery), важен шаринг по ссылке и работа в браузере на любой ОС. Для сложных моделей, тяжёлых вычислений, больших объёмов и корпоративной безопасности лучше Power BI/Tableau.

</details>
