:::tip[Коротко]
**Metabase** и **Apache Superset** — бесплатные open-source BI, которые компания разворачивает у себя. **Metabase** — про простоту: вопросы к данным почти без SQL, быстрый старт. **Superset** — мощнее и гибче, но требует технической команды. Оба экономят на лицензиях ценой self-hosting.
:::

## Зачем это нужно

Не все компании платят за Tableau/Power BI — многие стартапы и tech-команды берут open-source. Знать, что это и когда применяют, полезно: такие инструменты часто встречаются в вакансиях продуктовых компаний и СНГ-стартапов.

## Metabase: для кого

Самый дружелюбный open-source BI. Сильные стороны:

- **Низкий порог входа** — нетехнические люди задают «вопросы» к данным через визуальный конструктор, почти без SQL.
- Быстрая установка, приятный интерфейс, авто-графики.
- Есть платный Cloud, если не хочется хостить самим.

Минус — упирается в потолок на сложных моделях и тяжёлой аналитике.

## Superset: для кого

Apache Superset — мощнее и для более технической аудитории:

- Богатая библиотека визуализаций, гибкая настройка.
- Опирается на **SQL** (встроенный SQL Lab) — аналитику со знанием SQL раздолье.
- Хорошо масштабируется на большие данные и облачные хранилища.

Минус — сложнее в развёртывании и поддержке, менее «из коробки», чем Metabase.

## Сравнение с проприетарными BI

| | Metabase / Superset | Tableau / Power BI |
|--|---------------------|--------------------|
| Цена | бесплатно (open-source) | платные лицензии |
| Хостинг | свой сервер (self-host) | вендор/облако |
| Порог входа | Metabase низкий, Superset средний | средний |
| Поддержка | сообщество (+ платный Cloud) | вендорская |
| Тонкая аналитика | скромнее (DAX/LOD нет) | мощнее |

## Self-hosting: что нужно

:::caution[Бесплатно по лицензии ≠ бесплатно по затратам]
Open-source не требует лицензий, но требует **ресурсов на эксплуатацию**: сервер (часто через Docker), обновления, бэкапы, мониторинг, безопасность доступа. Для компании это работа DevOps/инженеров. «Бесплатный» BI оплачивается временем команды — учитывай это при сравнении с облачными Tableau/Power BI.
:::

Минимально: контейнер Docker, база для метаданных, доступ к источникам данных, и кто-то, кто это поддерживает.

## Когда выбирать open-source

- Ограниченный бюджет на лицензии, есть техническая команда.
- Нужен полный контроль над данными (всё на своих серверах).
- Metabase — когда нужен простой self-service для нетехнических коллег.
- Superset — когда команда сильна в SQL и нужна гибкость на больших данных.

## Задачи для самопроверки

<details>
<summary>1. Стартапу нужен бесплатный BI, где менеджеры сами смотрят данные почти без SQL. Что предложить?</summary>

Metabase: низкий порог входа, визуальный конструктор «вопросов» к данным, быстрый старт, нетехнические пользователи справятся. Superset мощнее, но требует SQL и сложнее в поддержке — для «самообслуживания менеджеров» Metabase удобнее.

</details>

<details>
<summary>2. «Open-source BI бесплатен». В чём подвох для компании?</summary>

Нет затрат на лицензии, но есть затраты на эксплуатацию: сервер, развёртывание (Docker), обновления, бэкапы, безопасность, мониторинг — это время инженеров/DevOps. Общая стоимость владения может оказаться сопоставимой с облачным BI; «бесплатность» относится только к лицензии.

</details>
