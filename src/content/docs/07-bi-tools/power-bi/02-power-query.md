:::tip[Коротко]
Power Query — этап **Transform**: здесь данные чистятся и приводятся в порядок **до** попадания в модель. Это тот же Power Query, что [в Excel](/03-excel/05-power-query-basics/): записывает шаги (Applied Steps), повторяет их при обновлении. Правило — всю очистку делай тут, а не формулами DAX потом.
:::

## Зачем это нужно

Грязные данные в модели = неверные меры и сломанные связи. Power Query чистит источник один раз и воспроизводит шаги при каждом refresh. Это правильное место для переименований, типов, фильтрации и объединения таблиц.

## Источники данных

Кнопка **Get Data** → десятки коннекторов: Excel/CSV, папка файлов, SQL Server, PostgreSQL, веб, Google Sheets, облачные сервисы. После выбора открывается редактор Power Query (**Transform data**).

## Editor и Applied Steps

Справа — панель **Applied Steps**: история каждого твоего действия. Любой шаг можно отредактировать, переставить или удалить, и Power Query пересчитает результат. Это делает обработку воспроизводимой и прозрачной.

## Базовые трансформации

Всё кликами, шаги записываются:

- удалить/переименовать столбцы;
- задать **тип данных** (текст → число → дата) — критично для корректных мер;
- фильтровать строки, убрать дубликаты;
- разделить столбец, заменить значения, заполнить пропуски;
- развернуть/свернуть (pivot/unpivot).

:::caution[Чистка — в Power Query, а не в DAX]
Переименования, типы, фильтрацию делай на этапе Power Query. Если оставить «грязь» и латать её мерами DAX, модель раздуется и замедлится, а ошибки типов вылезут в самый неподходящий момент. Правило: трансформации — слева (Power Query), вычисления — справа (DAX).
:::

## M-язык (обзор)

Каждый шаг под капотом — код на языке **M**. Обычно его не пишут руками: хватает кликов. Но строка формул видна, и продвинутые шаги иногда правят прямо в M. Глубоко учить не нужно на старте.

## Merge и Append

Два способа объединять запросы (как в Excel и [SQL](/02-sql/11-set-operations/)):

| Операция | Что делает | Аналог SQL |
|----------|-----------|------------|
| **Merge** | соединяет таблицы по ключу (+ столбцы) | `JOIN` |
| **Append** | подклеивает строки одной под другую | `UNION ALL` |

Merge — подтянуть справочник; Append — сложить выгрузки за разные месяцы.

## Задачи для самопроверки

<details>
<summary>1. Столбец с суммой загрузился как текст, и меры считаются неверно. Где чинить?</summary>

В Power Query: задать столбцу тип «Число» (Change Type) до загрузки в модель. Чистка типов — задача этапа Transform, а не DAX. После исправления типа Applied Steps применит его при каждом обновлении автоматически.

</details>

<details>
<summary>2. Есть 12 одинаковых по структуре файлов-выгрузок за месяцы. Как собрать в одну таблицу?</summary>

Append (или Get Data → Folder, который объединяет файлы папки). Append подклеивает строки одной таблицы под другую — аналог `UNION ALL`. Merge здесь не подходит: он соединяет по ключу и добавляет столбцы, а не строки.

</details>
