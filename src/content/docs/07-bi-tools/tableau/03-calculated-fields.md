:::tip[Коротко]
Вычисляемое поле (calculated field) — новая величина из формулы: маржа, доля, флаг. Отдельно стоят **табличные вычисления** (table calculations) — они считаются **по уже отображённым в таблице данным** (running total, % от итога, ранг) и зависят от направления вычисления.
:::

## Зачем это нужно

В данных редко есть готовая нужная метрика. Calculated fields позволяют посчитать её прямо в Tableau, не трогая источник: профит = выручка − затраты, конверсия, категоризация сумм.

## Создание

Правый клик в Data pane → **Create Calculated Field**. Формула — синтаксис, похожий на Excel/SQL:

```
// маржа
[Sales] - [Cost]

// флаг крупного заказа
IF [Sales] > 2000 THEN "крупный" ELSE "мелкий" END

// доля (осторожно: это не table calc)
[Profit] / [Sales]
```

## Виды функций

| Группа | Примеры |
|--------|---------|
| Арифметика | `+ - * /`, `ROUND`, `ABS` |
| Логика | `IF/THEN/ELSE`, `CASE`, `IIF` |
| Строки | `LEFT`, `MID`, `CONTAINS`, `SPLIT` |
| Даты | `DATEDIFF`, `DATEPART`, `DATETRUNC`, `TODAY` |
| Агрегаты | `SUM`, `AVG`, `COUNTD` (distinct) |

## Quick table calculations

Табличные вычисления оперируют **результатом**, который уже на листе. Быстрый способ: правый клик по мере на полке → **Quick Table Calculation**:

- **Running Total** — нарастающий итог.
- **Percent of Total** — доля от суммы.
- **Difference** — разница с предыдущим.
- **Rank** — ранг.
- **Moving Average** — скользящее среднее.

:::caution[Table calc зависит от направления (Compute Using)]
Табличное вычисление считается «вдоль» таблицы, и результат зависит от того, **по какому направлению** оно движется (Compute Using: Table across / down / Pane). Один и тот же «% of Total» даст разные числа при разном направлении. Если цифры странные — первым делом проверь Compute Using.
:::

## Custom table calculations

Для тонкого контроля используют функции `WINDOW_*` и `LOOKUP`/`INDEX`:

```
// доля от итога вручную
SUM([Sales]) / TOTAL(SUM([Sales]))

// сравнение с предыдущим периодом
SUM([Sales]) - LOOKUP(SUM([Sales]), -1)
```

Это аналог [оконных функций SQL](/02-sql/09-window-functions/): считают относительно «окна» строк, а не схлопывают данные.

## Обычное поле vs табличное вычисление

| | Calculated field | Table calculation |
|--|------------------|-------------------|
| Когда считается | на уровне строк/агрегата источника | по данным, уже выведенным на лист |
| Пример | `[Sales]-[Cost]` | running total, % of total, rank |
| Зависит от вида таблицы | нет | да (направление) |

<details>
<summary>1. Нужен нарастающий итог выручки по месяцам. Обычное поле или table calc?</summary>

Табличное вычисление (Running Total): нарастающий итог считается по уже выведенным на лист помесячным значениям, а не на уровне строк источника. Проверь направление (Compute Using), чтобы итог накапливался вдоль времени, а не поперёк.

</details>

<details>
<summary>2. «% of Total» показывает не те доли. С чего начать диагностику?</summary>

С направления вычисления (Compute Using / Table across vs down vs Pane). Табличные вычисления считаются вдоль таблицы, и неверное направление даёт неверные доли. Это самая частая причина «странных» процентов.

</details>
