:::tip[Коротко]
Дашборд — это сборка нескольких листов (sheets) в один интерактивный экран. Раскладку делай через **tiled** (сетка, рекомендуется), а интерактив — через [actions](/07-bi-tools/tableau/05-parameters-filters/). Для скорости минимизируй число листов и тяжёлых вычислений на одном дашборде.
:::

## Зачем это нужно

Отдельный график отвечает на один вопрос; дашборд связывает их в инструмент, где пользователь сам исследует данные. Умение собирать понятный быстрый дашборд — то, ради чего учат Tableau.

## Sheets vs Dashboards vs Stories

| Объект | Что это |
|--------|---------|
| **Sheet** (worksheet) | один график |
| **Dashboard** | несколько листов + фильтры на одном экране |
| **Story** | последовательность дашбордов/листов как презентация-нарратив |

Story — для пошагового [storytelling](/06-visualization/04-storytelling/); рабочая лошадка — Dashboard.

## Layout: tiled vs floating

- **Tiled** — элементы в сетке, не накладываются, авто-подгоняются. Рекомендуется по умолчанию: предсказуемо и адаптивно.
- **Floating** — свободное позиционирование с наложением. Гибко, но легко «разъедется» на другом размере экрана. Применяй точечно (например, плавающий заголовок).

## Actions

Делают дашборд интерактивным (Dashboard → Actions):

- **Filter** — выбор на одном листе фильтрует остальные.
- **Highlight** — подсветка связанных меток.
- **URL** — переход по ссылке.

## Мобильный лейаут

Tableau позволяет задать отдельную раскладку под телефон/планшет (Device Preview). На десктопе и мобильном дашборд может выглядеть по-разному — для мобильных оставляют ключевые виджеты вертикально. Если аудитория смотрит с телефона, проверь мобильный layout отдельно.

## Советы по производительности

:::tip[Как ускорить дашборд]
- **Меньше листов на дашборде** — каждый лист = отдельные запросы.
- **Extract вместо Live** на медленной БД ([подробнее](/07-bi-tools/tableau/02-connecting-data/)).
- **Фильтруй данные на источнике**, не тащи лишнее.
- Избегай тяжёлых table calc и большого числа меток (марок) на одном графике.
- Используй **context-фильтры**, чтобы сократить объём до основных вычислений.
:::

<details>
<summary>1. Tiled или floating для основной раскладки дашборда?</summary>

Tiled. Элементы выстраиваются в сетку, не накладываются и адаптируются к размеру — дашборд предсказуемо выглядит на разных экранах. Floating оставь для отдельных случаев (плавающий заголовок/легенда), иначе верстка «разъедется».

</details>

<details>
<summary>2. Дашборд медленно открывается. Три первых шага?</summary>

(1) Уменьшить число листов на дашборде — каждый порождает запросы. (2) Перейти с Live на Extract, если источник медленный. (3) Сократить объём данных фильтрами на источнике/context-фильтрами и убрать тяжёлые table calc.

</details>
