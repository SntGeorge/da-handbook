:::tip[Коротко]
Готовый дашборд нужно опубликовать. Для **портфолио** — Tableau Public (бесплатно, публичная ссылка в резюме). Внутри **компании** — Tableau Cloud (облако) или Server (self-hosted) с контролем доступа. Встраивание в сайт/продукт — embedded analytics.
:::

## Зачем это нужно

Дашборд в локальном файле никто не увидит. Публикация превращает его в ссылку, которой делятся: для junior это в первую очередь способ показать работу рекрутеру.

## Tableau Public — для портфолио

Бесплатная публикация в облако Tableau Public:

- **File → Save to Tableau Public** → работа публикуется, получаешь ссылку.
- Профиль на [public.tableau.com](https://public.tableau.com/) = твоё портфолио дашбордов.
- Можно встроить (embed) на личный сайт.

:::caution[Tableau Public — всё публично]
Любой дашборд на Tableau Public виден всем в интернете, и данные тоже. **Никогда не публикуй туда реальные/чувствительные данные компании** — только обезличенные или открытые наборы. Для пет-проекта бери публичные датасеты.
:::

## Tableau Cloud / Server — для компании

| Вариант | Где живёт | Для кого |
|---------|-----------|----------|
| **Tableau Cloud (Online)** | в облаке Tableau | без своей инфраструктуры |
| **Tableau Server** | на серверах компании | строгий контроль данных, on-premise |

Дают приватность, управление доступом (кто какие дашборды видит), расписания обновления экстрактов и [row-level security](/07-bi-tools/power-bi/07-publishing/) (разные пользователи видят разные строки).

## Embedded analytics (обзор)

Дашборды Tableau можно встраивать в веб-приложения и порталы (iframe или JavaScript API), чтобы аналитика жила внутри продукта компании. Это уже инженерная задача; аналитику достаточно знать, что такая возможность есть и что доступом управляет Server/Cloud.

## Задачи для самопроверки

<details>
<summary>1. Куда выложить дашборд для резюме и о чём помнить?</summary>

На Tableau Public — бесплатно, даёт публичную ссылку и профиль-портфолио. Главное: всё там общедоступно, поэтому используй только открытые/обезличенные данные, никогда не реальные данные работодателя.

</details>

<details>
<summary>2. Нужно, чтобы менеджеры разных регионов видели только свои данные в одном дашборде. Что использовать?</summary>

Публикацию в Tableau Cloud/Server с Row-Level Security: один дашборд, но строки фильтруются по пользователю/региону. Tableau Public для этого не годится — там нет приватности и управления доступом.

</details>
