:::tip[Коротко]
Главное деление ML — есть ли в данных **правильные ответы (метки)**. **Supervised** (с учителем): учим на примерах с ответами — прогноз, классификация. **Unsupervised** (без учителя): ответов нет, модель сама ищет структуру — кластеризация, снижение размерности. От этого зависит и выбор модели, и способ оценки.
:::

## Зачем это нужно

Это первый вопрос при любой ML-задаче: «есть ли у меня размеченные данные?». От ответа зависит, какие модели применимы и как мерить качество. Путаница здесь — частая ошибка на собеседовании.

## Supervised: с разметкой

Есть **признаки (X) и правильный ответ (y)**. Модель учится по парам «вход → ответ» и предсказывает y для новых данных.

- **Регрессия** — y число (выручка, цена).
- **Классификация** — y категория (отток да/нет, тип клиента).
- Пример: история заказов с меткой «вернулся/ушёл» → модель предсказывает отток новых.

Качество меряется сравнением предсказаний с известными ответами ([model evaluation](/10-ml-basics/08-model-evaluation/)).

## Unsupervised: без разметки

Есть только **признаки (X)**, ответов нет. Модель ищет структуру сама:

- **Кластеризация** — группирует похожих ([clustering](/10-ml-basics/06-clustering/)): сегменты клиентов без заранее заданных меток.
- **Снижение размерности** (PCA) — сжимает много признаков в несколько, сохраняя суть.
- Пример: разбить пользователей на поведенческие группы, не зная заранее, какие они.

:::caution[Без меток нет «правильного» ответа для проверки]
В unsupervised нельзя посчитать accuracy — не с чем сравнивать. Качество кластеризации оценивают косвенно (силуэт, интерпретируемость, бизнес-польза) и часто субъективно. Это принципиальное отличие: «модель нашла 5 кластеров» ещё не значит, что они осмысленны.
:::

## Сравнение

| | Supervised | Unsupervised |
|--|------------|--------------|
| Данные | X + ответы y | только X |
| Задачи | регрессия, классификация | кластеризация, PCA |
| Оценка | сравнение с y (accuracy, RMSE) | косвенная, субъективная |
| Пример | прогноз оттока | сегментация клиентов |

## Semi-supervised и reinforcement (обзор)

- **Semi-supervised** — размечена лишь часть данных (разметка дорогая). Модель использует и метки, и структуру неразмеченного. Компромисс между двумя мирами.
- **Reinforcement learning** — агент учится через **награды** за действия (игры, робототехника, динамическое ценообразование). В работе аналитика встречается редко; достаточно знать, что это отдельная парадигма «обучения методом проб и наград».

<details>
<summary>1. Есть история клиентов с меткой «ушёл/остался» — нужно предсказывать отток новых. Supervised или нет?</summary>

Supervised (классификация): у тебя есть признаки и правильные ответы (ушёл/остался), модель учится на них и предсказывает класс для новых клиентов. Качество проверяется сравнением предсказаний с фактом на отложенной выборке.

</details>

<details>
<summary>2. Почему для кластеризации нельзя посчитать accuracy?</summary>

Потому что нет «правильных» меток, с которыми сравнивать — это unsupervised. Модель сама придумала группы, эталона нет. Качество оценивают косвенно (метрики вроде силуэта, осмысленность и польза кластеров для бизнеса), и это во многом субъективно.

</details>
