:::tip[Коротко]
Линейная регрессия предсказывает **число** прямой $y = \beta_0 + \beta_1 x + \dots$, подбирая коэффициенты так, чтобы минимизировать ошибку (MSE). Её главная ценность для аналитика — **интерпретируемость**: каждый $\beta$ говорит, на сколько меняется $y$ при росте признака на единицу. Это та же регрессия, что и в [статистике](/05-statistics/08-correlation-regression/), но взгляд со стороны ML.
:::

## Зачем это нужно

Это самая объяснимая модель и часто — достаточная. Прежде чем тянуться к бустингу, аналитик пробует линейную регрессию: она быстро обучается, легко интерпретируется и служит бейзлайном, с которым сравнивают сложные модели.

## Уравнение, MSE, OLS

Модель ищет прямую (гиперплоскость):

$$\hat{y} = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \dots$$

Коэффициенты подбираются методом **наименьших квадратов (OLS)** — минимизируя средний квадрат ошибки:

$$\text{MSE} = \frac{1}{n}\sum (y_i - \hat{y}_i)^2$$

Квадрат штрафует большие промахи сильнее — поэтому регрессия чувствительна к выбросам.

## Интерпретация коэффициентов

Главное преимущество: $\beta_i$ = **на сколько изменится $y$ при росте $x_i$ на единицу, при прочих равных**.

```python
from sklearn.linear_model import LinearRegression
model = LinearRegression().fit(X, y)
model.coef_        # коэффициенты β
model.intercept_   # β0
```

Например, $\beta = 3$ при `ad_spend` → каждый рубль рекламы ассоциирован с +3 ₽ выручки (в рамках модели; [причинность это не доказывает](/05-statistics/08-correlation-regression/)).

## Допущения

:::caution[Линейная регрессия делает сильные предположения]
Метод корректен, когда выполняются допущения: **линейность** связи, независимость наблюдений, постоянная дисперсия ошибок (гомоскедастичность), нормальность остатков, отсутствие сильной **мультиколлинеарности** (коррелирующих признаков). Если связь нелинейна или признаки дублируют друг друга — коэффициенты врут, и нужна другая модель или преобразование данных.
:::

## Регуляризация: L1 и L2

Когда признаков много, модель переобучается. Регуляризация добавляет штраф за большие коэффициенты:

| Тип | Метод | Эффект |
|-----|-------|--------|
| **L2** | Ridge | сжимает коэффициенты, борется с мультиколлинеарностью |
| **L1** | Lasso | **обнуляет** часть коэффициентов → отбор признаков |
| L1+L2 | ElasticNet | комбинация |

Lasso удобен, когда нужно автоматически выкинуть бесполезные признаки.

## Когда не работает

- Связь существенно **нелинейна** (тогда — деревья/бустинг или фичеинжиниринг).
- Много **выбросов** (квадрат ошибки их раздувает).
- Сильная **мультиколлинеарность** без регуляризации.
- Сложные взаимодействия признаков, которые прямая не ловит.

<details>
<summary>1. В чём ценность линейной регрессии для аналитика, если есть точный бустинг?</summary>

Интерпретируемость и простота. Коэффициенты прямо говорят, как каждый фактор влияет на результат — это объяснимо бизнесу, в отличие от «чёрного ящика» бустинга. Плюс она быстрая и служит бейзлайном: если простая модель уже хороша, сложная может быть лишней.

</details>

<details>
<summary>2. Чем Lasso (L1) отличается от Ridge (L2) на практике?</summary>

Ridge сжимает все коэффициенты к нулю, но не обнуляет — хорош против мультиколлинеарности. Lasso может **обнулить** часть коэффициентов полностью, фактически отбирая признаки. Если нужно автоматически убрать бесполезные переменные — берут Lasso; если просто стабилизировать — Ridge.

</details>
