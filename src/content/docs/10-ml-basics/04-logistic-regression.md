:::tip[Коротко]
Несмотря на слово «регрессия», это модель **классификации**: она предсказывает **вероятность** класса (от 0 до 1) через сигмоиду. Порог (обычно 0.5) превращает вероятность в ответ да/нет. Как и [линейная](/10-ml-basics/03-linear-regression/), хорошо интерпретируется — через odds ratio — и часто служит надёжным бейзлайном для классификации (отток, дефолт, спам).
:::

## Зачем это нужно

Бинарная классификация — частая задача аналитика: уйдёт клиент или нет, одобрить кредит или нет. Логистическая регрессия даёт не просто метку, а **вероятность**, остаётся объяснимой и редко переобучается — отличная отправная точка.

## Сигмоида и log-odds

Линейная комбинация признаков «прогоняется» через сигмоиду, которая сжимает любое число в диапазон (0, 1) — это и есть вероятность:

$$p = \frac{1}{1 + e^{-(\beta_0 + \beta_1 x_1 + \dots)}}$$

Под капотом модель линейна по **log-odds** (логарифму шансов): $\log\frac{p}{1-p} = \beta_0 + \beta_1 x_1 + \dots$

## Log-loss

Обучается не на MSE, а на **log-loss** (логистическая функция потерь): она сильно штрафует уверенные, но неверные предсказания (сказал «99% уйдёт», а клиент остался). Это правильная метрика ошибки для вероятностей.

## Интерпретация: odds ratio

Коэффициент $\beta_i$ интерпретируется через **отношение шансов**: $e^{\beta_i}$ показывает, во сколько раз меняются шансы при росте признака на единицу.

- $e^{\beta} = 1.5$ → признак увеличивает шансы класса на 50%.
- $e^{\beta} < 1$ → снижает шансы.

Это понятнее бизнесу, чем абстрактные веса, — сильная сторона метода.

## Подбор порога (threshold)

:::caution[Порог 0.5 — не догма]
Модель выдаёт вероятность; решение да/нет зависит от **порога**. По умолчанию 0.5, но его двигают под задачу: если пропустить отток дорого — снижают порог (ловим больше потенциальных уходов ценой ложных тревог). Выбор порога — это баланс [precision и recall](/10-ml-basics/08-model-evaluation/), а не свойство модели.
:::

```python
from sklearn.linear_model import LogisticRegression
model = LogisticRegression().fit(X, y)
proba = model.predict_proba(X_new)[:, 1]    # вероятности класса 1
pred = (proba > 0.3).astype(int)             # свой порог
```

## Multinomial

Для **более чем двух** классов есть многоклассовый вариант (multinomial / softmax): модель предсказывает вероятности по всем классам, выбирается самый вероятный. Бинарный случай — частный.

<details>
<summary>1. Почему «регрессия», если задача — классификация?</summary>

Потому что под капотом модель линейно предсказывает log-odds (логарифм шансов) — это «регрессия» в математическом смысле. Сигмоида затем превращает результат в вероятность, а порог — в класс. Историческое название вводит в заблуждение: по задаче это классификатор.

</details>

<details>
<summary>2. Модель ловит мало реальных оттоков, хотя точность высокая. Что покрутить?</summary>

Снизить порог классификации (с 0.5, например, до 0.3): тогда больше клиентов попадёт в «уйдёт», recall по оттоку вырастет ценой большего числа ложных тревог. Это компромисс precision/recall, настраиваемый порогом, а не переобучение модели. Цена пропуска оттока определяет, куда двигать порог.

</details>
