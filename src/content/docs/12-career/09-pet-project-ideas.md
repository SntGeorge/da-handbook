:::tip[Коротко]
Хороший pet-проект показывает **полный цикл аналитика**: вопрос → данные → анализ → вывод → визуализация. Лучшие идеи — на данных, которые тебе **интересны лично** (твоя музыка, финансы, любимая игра): мотивация выше, и на собесе ты горишь рассказом. Один доведённый до конца проект ценнее десяти набросков.
:::

## Зачем это нужно

[Портфолио](/12-career/02-portfolio/) без проектов пустое. Эти идеи дают конкретику: что взять, чтобы продемонстрировать SQL, Python, BI и продуктовое мышление. Выбирай то, что зацепит лично, — так доведёшь до конца.

## Как выбрать и оформить проект

- **Один сквозной вопрос**, на который проект отвечает (не «поиграл с данными», а «влияет ли X на Y»).
- Полный цикл: сбор → [очистка](/04-python/10-data-cleaning/) → анализ → визуализация → **вывод**.
- README с целью, данными и результатами (см. [портфолио](/12-career/02-portfolio/)).
- По возможности — живой дашборд ([Tableau Public/Looker](/07-bi-tools/looker/01-intro/)).

## Публичные данные

Готовые датасеты — быстрый старт:

- **Kaggle** — тысячи датасетов на любую тему.
- **Открытые госданные** — демография, транспорт, погода.
- Классические учебные базы: продажи, такси NYC, авиарейсы.
- [Когортный анализ](/08-product-analytics/04-cohort-analysis/) или RFM на открытом датасете e-commerce.

## Свои данные (самые мотивирующие)

- **Spotify / Last.fm** — анализ своей музыки за год (жанры, сезонность прослушиваний).
- **Apple Health / шагомер** — личная активность, сон, корреляции.
- **Личные финансы** — дашборд расходов по категориям и месяцам.
- **Переписка/чат** — частота, время суток, активность (с заботой о приватности).

## Скрейпинг + анализ

Показывает [работу с API/вебом](/04-python/15-apis-and-scraping/):

- **Парсер вакансий** + анализ требуемого стека (живая связь с [обзором рынка](/00-intro/market-stack-2026/)).
- Цены товаров/недвижимости во времени.
- Анализ Telegram-канала через API (активность, темы, рост).

:::caution[Скрейпь этично и без персональных данных]
Соблюдай `robots.txt` и rate limits, не выкладывай в публичный репозиторий персональные данные и не коммить токены/ключи ([см. этику скрейпинга](/04-python/15-apis-and-scraping/)). Аккуратность с данными — это и есть сигнал зрелого аналитика, который оценят рекрутеры.
:::

## Симуляции и продуктовые проекты

- **Симуляция A/B-теста**: сгенерировать данные, проверить значимость, показать [подводные камни](/09-ab-testing/07-common-pitfalls/) (peeking, SRM).
- **Воронка и retention** на событийных данных.
- Сегментация клиентов через [кластеризацию](/10-ml-basics/06-clustering/) или RFM.
- Прогноз простой метрики ([регрессия](/10-ml-basics/03-linear-regression/)).

## Таблица идей по навыкам

| Проект | Что демонстрирует |
|--------|-------------------|
| Дашборд личных финансов | BI, визуализация |
| Парсер вакансий + стек | API/скрейпинг, Python, выводы |
| Когорты/RFM на e-commerce | SQL, продуктовая аналитика |

## Задачи для самопроверки

<details>
<summary>1. Что превращает «поигрался с датасетом» в сильный pet-проект?</summary>

Чёткий вопрос и полный цикл с выводом: проект должен отвечать на конкретный вопрос (влияет ли X на Y, где теряем пользователей) и доходить до визуализации и осмысленного заключения, оформленного в README. Набор разрозненных графиков без вопроса и вывода впечатления не производит.

</details>

<details>
<summary>2. Не знаешь, какой проект выбрать. По какому критерию решать?</summary>

По личному интересу к данным (своя музыка, финансы, игра, тема, которая нравится). Мотивация резко повышает шанс довести проект до конца, а на собеседовании ты увлечённо его рассказываешь — это считывается. Доведённый до конца проект на интересных данных бьёт «правильный, но скучный», который забросишь.

</details>
